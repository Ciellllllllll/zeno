# Visual Studio 2022 SDK Use

Use Visual Studio 2022 17.7 or newer with MSVC v143 and a Windows SDK. The repository and packaged SDK roots use CMakePresets schema version 6.

## Build The SDK Package From The Repository

Open a Developer PowerShell or a normal PowerShell where Cargo and CMake are available:

```powershell
.\scripts\package-sdk.ps1
```

The package is written to:

```text
build/package-sdk/ZenoEngine-SDK-v0.1.0-dev/
```

The repository root also exposes the package target through CMake. This target creates both Debug and Release SDK artifacts:

```powershell
cmake --build --preset windows-msvc-debug --target zeno_package_sdk
cmake --build --preset windows-msvc-release --target zeno_package_sdk
```

In Visual Studio 2022 Open Folder, open the repository root, select `windows-msvc-debug` or `windows-msvc-release`, and build the `zeno_package_sdk` target. The target produces the same Debug/Release package layout from either active preset.

## Open The Packaged Template

1. In Visual Studio 2022, choose Open Folder.
2. Open `build/package-sdk/ZenoEngine-SDK-v0.1.0-dev/templates/cpp_empty`.
3. Select `windows-msvc-debug` or `windows-msvc-release` from the packaged template presets.

The template uses `find_package(ZenoEngine CONFIG REQUIRED)` and links `ZenoEngine::zeno_sdk_cpp`.
The packaged preset sets `ZenoEngine_DIR` to the SDK `cmake` directory and the post-build step copies the matching `bin/<Config>/zeno_abi.dll` beside the executable.

## Open The Packaged Samples

Open:

```text
build/package-sdk/ZenoEngine-SDK-v0.1.0-dev/samples/sdk_feature_samples_cpp
```

Select `windows-msvc-debug` or `windows-msvc-release` from the packaged sample presets. The sample executables are windowed, so run them only from a local desktop session where opening a Win32 window is acceptable.

## Repository Builds

When working inside the repository, Visual Studio should open the repository root as a folder and use `CMakePresets.json`. The canonical presets are:

- `windows-msvc-debug`
- `windows-msvc-release`

Generated `.sln` and `.vcxproj` files are not source-of-truth build inputs.
